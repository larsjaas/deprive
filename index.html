About

Deprive is a command line utility for doing object file interdependency analysis. The primary audience is software developers involved in development on largescale C/C++ projects.

Basic Usage

Given a project that builds all its object files under src/ and has a main source file named ls.c (for this example I have been using fileutils-4.1), you can show the dependency graph for the selected program like this: 

$ deprive --scan --analyze lib/*.o src/*.o
$ deprive --trace ls.o --digraph | dot -Tpng - >ls.png

The above commands will produce the following ls.png image (follow image link to get the full size image):

From this image you can see that ls.o references argmatch.o, while argmatch.o also references ls.o, so you have a circular reference. Inspecting the object files, you will find out that ls.o defines the function "usage", which argmatch.o imports for some reason. There is also a circular reference from ls.o via human.o to argmatch.o in the graph.

Other Usage

The dependency database can trivially be loaded directly into Python, so there are lots of combination possibilities there. You can for instance use the dependency tree from SCons to automatically link up unittests programs so you do not have to set up and maintain the source file list for each unittest program manually.

Deprive Dependencies

Deprive is implemented in Python. To view the dependency graphs graphically you will also need GraphViz (dot) to render the graph descriptions.
